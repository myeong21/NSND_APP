# 🚗 DrowsyGuard — 주차별 개발 계획 요약

> **팀 인원:** 4명 | **전체 기간:** 5주 (1주차 환경 구축 + 2~5주차 백엔드 개발)

---

## 📅 주차별 목표 한눈에 보기

| 주차 | 목표 | 핵심 작업 | 완성 기능 |
|:----:|------|-----------|:---------:|
| **1주차** | 기반 구축 | 딥러닝 모델 완성, 앱 UI/UX 설계, Tailscale·GitHub 환경 구축 | 🛠️ 환경 |
| **2주차** | 인증 + SMS | FastAPI 서버 세팅, 로그인/회원가입 API, 앱 네이티브 SMS 발송 | 기능 ①② |
| **3주차** | 실시간 감지 | LSTM→FastAPI 통합, WebSocket 실시간 전송, 3단계 졸음 판정, DB 로그 저장 | 기능 ③ |
| **4주차** | 리포트 + 통합 | 리포트 API·시각화(fl_chart), 안전 점수, 통합 테스트 | 기능 ④ |

---

## ⚡ 졸음 단계별 경보

| 단계 | 조건 | 행동 |
|:----:|------|------|
| 1단계 | `pred > 0.5` × 10프레임 | 앱 경보음 + 진동 |
| 2단계 | `pred > 0.7` × 15프레임 | *(미정)* |
| 3단계 | `pred > 0.85` × 20프레임 | **자동 문자 발송** |

## 🛠️ 기술 스택 요약

| 영역 | 기술 |
|------|------|
| 백엔드 | FastAPI (Python) + uvicorn |
| 데이터베이스 | SQLite → PostgreSQL |
| 인증 | JWT + bcrypt |
| 실시간 통신 | WebSocket |
| 딥러닝 모델 | LSTM (`best_drowsy_model.keras`) |
| 모바일 앱 | Flutter (Android / iOS) |
| SMS (Android) | `telephony` 패키지 — 자동 발송 |
| SMS (iOS) | `url_launcher` — 반자동 발송 |
| 차트 | `fl_chart ^0.68.0` |
| 네트워킹 | Tailscale VPN |
| 버전 관리 | GitHub (feature branch 전략) |

---

## 📋 GitHub 브랜치 전략

```
main
└── develop
    ├── feature/auth        ← 인증 기능
    ├── feature/sms         ← 문자 발송
    ├── feature/detection   ← 실시간 감지
    └── feature/report      ← 리포트
```

> PR은 최소 1명 리뷰 후 `develop` 머지. 매주 금요일 `develop → main` 머지 후 Jetson 배포.
